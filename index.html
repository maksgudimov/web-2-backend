1) получить главную страницу методом GET в протоколе HTTP 1.0

Мы хотим получить содержимое страницы, использую метод GET. HTTP / 1.0 может поддерживать длинные соединения HTTP / 1.0 считает, что каждый сервер привязан к уникальному ip, поэтому заголовок запроса не содержит имени хоста (поле HOST)

2) получить внутреннюю страницу методом GET в протоколе HTTP 1.1

HTTP / 1.1 несколько запросов и ответов могут выполняться одновременно, т.е. конвейерная обработка HTTP / 1.1 добавляет поле HOST HTTP / 1.1 добавляет код состояния 100. Использование кода состояния 100 (Продолжить) позволяет клиенту протестировать сервер с заголовком запроса перед отправкой тела сообщения запроса, чтобы узнать, хочет ли сервер получить тело запроса, а затем решить, отправлять ли тело запроса.
